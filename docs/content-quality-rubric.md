# WeWrite 内容质量验收

目标不是让文字“像人”，而是让成稿在事实、判断和读者收益上可以直接使用。

## 五项判断

每项 1-5 分：准确、观点、有用、合声、好读。通过条件是平均分至少 4、单项不低于 3，且
没有以下阻断问题：虚构作者经历；核心事实无来源；结论超出证据；没有明确读者交付。

审稿决定只有三种：`pass`、`revise`、`needs_input`。`revise` 必须实际改稿再复审，不能只给
建议；`needs_input` 仅用于用户明确要求个人故事但关键材料缺失的情况。

## 可追溯产物

- `brief.yaml`：读者、问题、交付、核心判断、反方、边界和个人材料可用性。
- `claims.yaml`：事实、推断、意见、用户经历及其来源状态。
- `draft.md`：未经编辑通过的初稿。
- `article.md`：通过编辑门槛后的成稿。
- `review-report.json`：五项分数、阻断问题、编辑决定和初稿到成稿修改幅度。

## 迭代观察指标

- `publishable`：是否达到交付门槛，必须为 true 才能封存任务。
- `edit_ratio`：初稿到成稿的文本变化，用于观察初稿稳定性，不设单篇硬门槛。
- `structure_changed`：审稿是否调整了章节结构。
- 人工后续修改率：在有真实发布编辑数据后持续观察，目标逐步降到 15% 以内；没有样本时不
  宣称已经达到。

语言风险分只用于发现套话、碎句和节奏过齐，不参与 `publishable` 的硬判断。
